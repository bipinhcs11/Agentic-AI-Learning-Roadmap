**🧠 First — what even is an LLM?**

A Large Language Model (LLM) is an AI trained on massive text datasets that can generate, summarise, translate, and reason about language. Think ChatGPT — but you can run one locally on your own machine using Ollama.

No cloud. No API key. No usage bill. Full control.

---

**📚 What is RAG — and why does every enterprise app need it?**

RAG = Retrieval-Augmented Generation.

The problem with vanilla LLMs in enterprise: they only know what they were trained on. They hallucinate. They can't access your internal docs, your Confluence pages, your policy manuals.

RAG fixes this by:
1. Taking your documents → converting them to numerical vectors (embeddings)
2. When a user asks a question → finding the most relevant chunks from *your* data
3. Sending those chunks to the LLM as context → getting answers grounded in *your* knowledge

Result: an AI that answers from your actual company data, not from guesses.

Use cases: internal knowledge base search, contract review, HR policy Q&A, code documentation assistant, support ticket resolution.

**📍 What I built across 4 phases:**

**Phase 1 — Foundation** ✅
Set up a fully local AI stack on Mac Mini M4.
First chat agent, REST API basics, prompt engineering.

**Phase 2 — RAG System** ✅
Built a document Q&A system from scratch.
Embeddings with nomic-embed-text, cosine similarity search, chunking strategies.
No PyTorch, no cloud — pure Ollama + Python.

**Phase 3 — Agentic Stack** ✅
Agents that reason and use tools (calculator, web scraper, file system).
Persistent memory across sessions.
LLM-as-Judge evaluation framework to score answer quality.
Wrapped it all in a FastAPI server.

**Phase 4 — Agent Framework** ✅
Built my own mini-LangChain from scratch.
OpenAI-compatible API (swap OpenAI → local model with 2 lines of code).
Streamlit chat UI running 100% locally.
Full platform combining everything.
