# RAG Path

Use this path if your main goal is retrieval, document intelligence, citations,
or evaluation.

## Recommended Order

| Step | Folder | Skill |
|---:|---|---|
| 1 | `Phase2_RAG_Systems/project_01_first_rag/` | Build chunks, embeddings, similarity search, and generation manually |
| 2 | `Phase2_RAG_Systems/project_04_multi_doc_rag/` | Move from toy docs to multiple documents |
| 3 | `Phase2_RAG_Systems/project_05_agentic_rag/` | Let an agent decide when and how to retrieve |
| 4 | `Phase3_Agentic_Stack/project_05_rag_evaluation/` | Evaluate answer quality |
| 5 | `Phase7_Advanced_AI_Patterns/project_01_graphrag/` | Add graph structure to retrieval |
| 6 | `Phase9_Dynamic_Agentic_RAG_MCP/capstone_enterprise_assistant_hub/` | Combine tenant RAG with MCP tools and provider routing |

## What To Watch For

- Does the README name the required local models?
- Is retrieval using embeddings, lexical fallback, or both?
- Does the answer cite the retrieved source?
- Does the project explain what happens when Ollama is offline?

## Good Portfolio Outcome

Build a small domain-specific assistant over your own safe sample documents, then
show the architecture, retrieval behavior, expected output, and failure modes.
